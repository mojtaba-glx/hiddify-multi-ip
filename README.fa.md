# 🌐 Hiddify Multi-IP Outbound

> هر آی‌پی سرور رو از **خودش** به اینترنت وصل کن — کاملاً خودکار. ساخته‌شده برای **هیدیفای منیجر نسخه ۱۰**.

اگه سرورت **چند آی‌پی عمومی** داره (مثلاً آلمان + هلند + لهستان)، هیدیفای به‌صورت پیش‌فرض **همه‌ی ترافیک خروجی رو فقط از آی‌پی اصلی** می‌فرسته. این اسکریپت این مشکل رو حل می‌کنه: هر کانکشنی که روی آی‌پی `X` میاد، از همون آی‌پی `X` خارج می‌شه — خودکار و پایدار.

---

## ✨ امکانات

- 🧠 **تشخیص خودکار و بدون تنظیم** — همه‌ی آی‌پی‌های سرور، آی‌پی اصلی، و کشور هرکدوم رو با GeoIP پیدا می‌کنه
- 🔁 **پایدار و خود‌ترمیم‌گر** — بعد از `Apply` پنل، ریبوت، و آپدیت هیدیفای سرپا می‌مونه (systemd path-watcher + تایمر ۳۰ ثانیه‌ای)
- ⚡ **بدون قطعی در مصرف عادی** — ری‌استارت هوشمند مبتنی بر hash: سرویس‌ها فقط وقتی ری‌استارت می‌شن که کانفیگ *واقعاً* عوض بشه (دیگه قطعی ۳۰ ثانیه‌ای تو بازی آنلاین نداری)
- 🔢 **پشتیبانی از N آی‌پی** — نه فقط ۲ تا؛ هر چند تا آی‌پی اضافه که سرورت داره
- 🧹 **حذف تمیز** — `--remove` همه‌ی تزریق‌ها رو جراحی‌وار پاک می‌کنه (بدون وابستگی به بک‌آپِ شکننده)
- 🛡️ **امن** — از فایل‌های اصلی بک‌آپ می‌گیره، دیتابیس هیدیفای رو دست نمی‌زنه، کاملاً قابل برگشت

## 🚀 نصب (یک خط)

```bash
bash <(curl -sSL https://raw.githubusercontent.com/YOUR_USER/YOUR_REPO/main/hiddify-multi-ip.sh)
```

> بعد از انتشار، `YOUR_USER/YOUR_REPO` رو جایگزین کن. برای تست محلی کافیه:
> ```bash
> bash hiddify-multi-ip.sh
> ```

اسکریپت این کارها رو می‌کنه:
1. همه‌ی آی‌پی‌ها + کشورها (GeoIP) رو تشخیص می‌ده
2. دامنه‌های هیدیفای رو از `current.json` می‌خونه
3. inboundهای Xray + outboundهای `sendThrough` + قوانین routing رو تزریق می‌کنه
4. ACLهای تفکیک بر اساس `dst` رو به HAProxy اضافه می‌کنه
5. سرویس‌های خود‌ترمیم‌گر systemd رو نصب می‌کنه
6. اعمال و ری‌استارت

در پایان یه خلاصه می‌بینی:

```
  Default exit : 203.0.113.10
  nl     exit   : 198.51.100.20  (Netherlands)
  pl     exit   : 192.0.2.30     (Poland)
```

```

## ⚙️ چطور کار می‌کنه

هیدیفای نسخه ۱۰ **HAProxy رو جلوی Xray** می‌ذاره و روی پورت `:443` همه‌ی آی‌پی‌ها رو جمع می‌کنه. مشکل اینجاست: Xray نمی‌تونه بفهمه کلاینت به کدوم آی‌پی مقصد وصل شده — پس همه‌ی ترافیک از آی‌پی اصلی خارج می‌شه.

این اسکریپت مشکل رو در دو لایه حل می‌کنه:

```
کلاینت → HAProxy https-in (:443، همه‌ی آی‌پی‌ها)
          │
          ├─ acl dst <آی‌پی_هلند>  → to_https_in_ssl_nl → in-tcpmode_nl
          ├─ acl dst <آی‌پی_لهستان> → to_https_in_ssl_pl → in-tcpmode_pl
          └─ پیش‌فرض               → to_https_in_ssl    → in-tcpmode
                                      │
                                      └→ inbound Xray (با تگ _nl / _pl / پیش‌فرض)
                                           │
                                           └→ routing: inboundTag=_nl → outbound multi_nl
                                                     (sendThrough = آی‌پی هلند)
```

یعنی ترافیکی که روی آی‌پی `X` میاد، تو HAProxy تگ می‌خوره، به یک inbound اختصاصی Xray می‌ره، و با `sendThrough` که به همون آی‌پی بایند شده از سرور خارج می‌شه.

## ❓ سؤالات متداول

**س: آی‌پیم کشور اشتباه نشون می‌ده (مثلاً آی‌پی «هلند» رو انگلیس نشون می‌ده)؟**
ج: این یک نادرستی پایگاه داده‌ی GeoIP از سمت ارائه‌دهنده‌ی هاستینگتونه — روی routing **تأثیری نداره**. آی‌پی خروجی همچنان درسته. می‌تونی از `config.env` برچسب رو عوض کنی.

**س: بعد از آپدیت هیدیفای سرپا می‌مونه؟**
ج: بله. systemd path-watcher می‌فهمه که هیدیفای کانفیگ‌ها رو بازنویسی کرده و ظرف چند ثانیه دوباره اعمال می‌کنه. تایمر ۳۰ ثانیه‌ای هم تور دوم امنیتیه.

**س: کانکشن‌های فعلی رو قطع می‌کنه؟**
ج: فقط وقتی کانفیگ *واقعاً* عوض بشه (مثلاً بعد از زدن `Apply`). مصرف عادی (بازی، استریم) تحت تأثیر قرار نمی‌گیره — قطعی ۳۰ ثانیه‌ای نداری.
